## Overview

The Commet plugin enables subscription-aware billing for your xmcp server using [Commet](https://commet.co/?utm_source=xmcp&utm_medium=docs&utm_campaign=plugin). Your tools get full context: which plan the customer is on, what features they can access, how much usage remains, and automatic consumption tracking.

- **Feature-level gating**: Tool A is free, Tool B is Pro, Tool C is Enterprise
- **Usage tracking with enforcement**: metered, credits, or balance models with real limits
- **AI token billing**: track per-model token consumption with configurable margins
- **Rich context**: your tools know the customer's plan, remaining quota, and limits
- **Full billing**: invoices, proration, checkout, customer portal
- **Taxes and compliance**: Commet handles everything as Merchant of Record

## Commet Setup

Follow these steps to configure your billing product before integrating the plugin:

1. **Create an account** at [commet.co/signup](https://commet.co/signup?utm_source=xmcp&utm_medium=docs&utm_campaign=plugin)
2. **Create a Product** from the dashboard. This represents your xmcp server
3. **Define your Plans** (e.g., Free, Pro, Enterprise). Each plan includes a set of features
4. **Add Features** to each plan. Choose the type per feature:
   - **Boolean**: on/off access (e.g., `export`, `custom-branding`)
   - **Metered**: usage-based with included quotas and optional overage pricing (e.g., `ai-generate` with 1000 included units)
5. **Set pricing** for each plan: monthly/yearly intervals, per-seat, or flat rate
6. **Copy your API Key** (`ck_xxx`) from Settings > API Keys

<Callout variant="info">
  Use the sandbox environment (`ck_test_xxx`) during development. Switch to your production key when you're ready to go live.
</Callout>

## Configuration

Register the Commet provider in your middleware:

```typescript title="src/middleware.ts"
import { commetProvider } from "@xmcp-dev/commet";

export default commetProvider({
  apiKey: process.env.COMMET_API_KEY!,
  environment: "sandbox",
});
```

### Configuration Options

```typescript
interface CommetProviderConfig {
  apiKey: string;
  environment?: "production" | "sandbox";
  customerHeader?: string;
}
```

- `apiKey`: Your Commet API key (starts with `ck_`)
- `environment`: Defaults to the SDK's default environment
- `customerHeader`: HTTP header name for the customer identifier (defaults to `"customer-key"`)

<Callout variant="info">
  Customer identity is provided via the `customer-key` header (or whatever you configure in `customerHeader`). This is the same ID you use when creating customers in Commet.
</Callout>

## Feature Gating

Use `check` to gate tools behind boolean features. No usage is tracked:

```typescript
import { check } from "@xmcp-dev/commet";

const result = await check("custom-branding");

if (!result.allowed) {
  return result.message;
}
```

### Check Response

```typescript
{
  allowed: boolean;
  code: "feature_enabled" | "feature_not_allowed" | "no_subscription";
  message: string;
  plan?: string;       // "Pro", "Enterprise", etc.
}
```

## Usage Tracking

Use `track` for consumption-based features. It checks access and records usage in one call:

```typescript
import { track } from "@xmcp-dev/commet";

const result = await track({
  feature: "ai-generate",
  units: 1,
});

if (!result.allowed) {
  return result.message;
}

// result.remaining -> 847
// result.included  -> 1000
```

For operations with different cost weights:

```typescript
const result = await track({
  feature: "ai-generate",
  units: 5,
});
```

### Track Response

```typescript
{
  allowed: boolean;
  code: "tracked" | "feature_not_allowed" | "no_subscription";
  message: string;
  plan?: string;         // "Pro"
  remaining?: number;    // 847
  included?: number;     // 1000
  overage?: number;      // 0
  unlimited?: boolean;   // false
}
```

## AI Token Tracking

For AI-powered tools, track token consumption per model:

```typescript
import { track } from "@xmcp-dev/commet";

const result = await track({
  feature: "ai-generate",
  model: "claude-sonnet-4-20250514",
  inputTokens: 1200,
  outputTokens: 340,
  cacheReadTokens: 500,
});
```

Commet calculates costs using the AI model catalog with configurable margins per feature.

## Helpers

### `getCustomerId()`

Returns the customer ID extracted from the request header. Throws if no customer header is present:

```typescript
import { getCustomerId } from "@xmcp-dev/commet";

const customerId = getCustomerId();
```

### `getPortalUrl()`

Returns the customer's billing portal URL for upgrade/management flows:

```typescript
import { getPortalUrl } from "@xmcp-dev/commet";

const portalUrl = await getPortalUrl();
```

## Example

A complete example with a free tool, a boolean-gated tool, and a metered tool:

```typescript title="src/middleware.ts"
import { commetProvider } from "@xmcp-dev/commet";

export default commetProvider({
  apiKey: process.env.COMMET_API_KEY!,
  environment: "production",
});
```

```typescript title="src/tools/free-tool.ts"
import { z } from "zod";
import { type InferSchema, type ToolMetadata } from "xmcp";

export const schema = {
  query: z.string().describe("The search query"),
};

export const metadata: ToolMetadata = {
  name: "search",
  description: "Search public documentation, free for all users",
};

export default async function search({ query }: InferSchema<typeof schema>) {
  return `Results for "${query}": [doc1, doc2, doc3]`;
}
```

```typescript title="src/tools/export-tool.ts"
import { z } from "zod";
import { type InferSchema, type ToolMetadata } from "xmcp";
import { check } from "@xmcp-dev/commet";

export const schema = {
  format: z.enum(["csv", "json", "pdf"]).describe("Export format"),
};

export const metadata: ToolMetadata = {
  name: "export",
  description: "Export data, Pro plan only",
};

export default async function exportData({
  format,
}: InferSchema<typeof schema>) {
  const result = await check("export");

  if (!result.allowed) {
    return result.message;
  }

  return `Exported data as ${format} (plan: ${result.plan})`;
}
```

```typescript title="src/tools/pro-tool.ts"
import { z } from "zod";
import { type InferSchema, type ToolMetadata } from "xmcp";
import { track } from "@xmcp-dev/commet";

export const schema = {
  prompt: z.string().describe("The prompt to generate content from"),
};

export const metadata: ToolMetadata = {
  name: "ai-generate",
  description: "Generate content with AI, metered feature that tracks units",
};

export default async function aiGenerate({
  prompt,
}: InferSchema<typeof schema>) {
  const result = await track({
    feature: "ai-generate",
    units: 1,
  });

  if (!result.allowed) {
    return result.message;
  }

  return `Generated: ${prompt} (${result.remaining ?? "unlimited"} uses remaining)`;
}
```
